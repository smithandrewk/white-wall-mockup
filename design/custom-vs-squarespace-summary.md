# Custom Site vs. Squarespace: Quick Summary for Drew & Max

**March 2026 — Prepared by Andrew**

---

## The Short Version

Your Squarespace site is 29 static pages, some images, and a hacky $0-product booking workaround. You don't use any Squarespace-specific features that can't be replaced for free. Your two external services — **Acuity Scheduling** and **Square Payments** — work on any platform. They're not tied to Squarespace.

Moving to a custom site hosted on Vercel or Cloudflare Pages would cost **~$10/year** (just the domain) vs. **$300-700/year** on Squarespace. But the real argument isn't cost — it's speed, flexibility, and protection.

---

## Why Custom Wins: The Claude Code Factor

I use an AI coding tool called Claude Code that can build, edit, and maintain websites extremely fast. Here's the difference:

| Task | On Squarespace | On a Custom Site (with Claude Code) |
|------|---------------|-------------------------------------|
| Change site-wide colors/fonts | Click through every section manually | One command, done in seconds |
| Add a new studio set page | Copy a page, manually edit every block | Duplicate a template, auto-populate |
| Redesign the gallery | Limited to Squarespace's gallery blocks | Full creative control, any layout |
| Add new features (chat, AI booking assistant, automations) | **Not possible** | Built and deployed same day |
| Fix something across all 29 pages | Edit each page one by one | One command updates all pages |
| Turnaround on a batch of changes | Days/weeks (manual work) | Hours (automated) |

**Bottom line: Every change you ask for on Squarespace costs me 3-5x the time it would on a custom site.** Claude Code can't touch Squarespace — it's a closed platform. On a custom site, I can make sweeping changes in minutes.

## ADA Compliance: Custom Is Actually Safer

You may have heard about serial ADA lawsuits — lawyers who scan websites with automated tools and sue small businesses over accessibility failures (missing image descriptions, poor color contrast, keyboard navigation issues). Over 4,000 federal lawsuits were filed in 2024 alone, and small businesses are the primary target.

| | Squarespace | Custom Site |
|--|-------------|-------------|
| **Baseline accessibility** | Templates have *some* built-in support | We build WCAG 2.1 AA compliance in from day one |
| **Control over fixes** | Limited — can't modify Squarespace's underlying code | Full control over every element |
| **Third-party widgets** | Squarespace injects code you can't audit or fix | We choose what goes on the page |
| **Automated scan results** | Often flags issues in Squarespace's own code that you can't fix | Every flag is something we can address |
| **Ongoing compliance** | Manual checks, no tooling | Claude Code can audit and fix accessibility issues across all pages in one pass |

**The risk of getting sued is the same on either platform** — serial plaintiffs don't care what you're built on. But your ability to *fix issues quickly and completely* is dramatically better on a custom site. On Squarespace, some accessibility problems are baked into the platform and you literally cannot fix them.

## What About Acuity & Square?

Neither service is tied to Squarespace:

- **Acuity Scheduling**: Works standalone. We embed it on the new site with one line of code. Zero disruption — same links, same booking flow, same everything.
- **Square Payments**: Processed externally. Works anywhere. No change needed.

Nothing breaks. Nothing migrates. They just keep working.

## What You'd Lose

One thing: **Squarespace's visual drag-and-drop editor.** If you frequently edit page content yourselves, you'd either:

1. Text me and I push the change (usually < 5 minutes with Claude Code)
2. We set up a simple CMS (free) so you can edit text/images in a browser

If you rarely edit the site yourselves (which seems to be the case), this doesn't matter.
